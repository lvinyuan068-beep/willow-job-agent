# 10｜技能与岗位匹配矩阵 v1.0

更新日期：2026-07-15  
判断标准：只记录已有证据或本人明确确认的能力，不用“接触过”替代“能独立交付”。

## 1. 能力等级

- L3 独立交付：无需系统教学即可完成，有本地作品或工作材料验证。
- L2 可执行：能在 AI、教程或明确流程辅助下完成，并能复盘。
- L1 概念/学习中：知道用途，但尚未形成可独立展示的结果。
- L0 未验证：当前材料不足，不在简历中主张。

## 2. 能力矩阵

| 能力 | 等级 | 证据 | 限制 |
|---|---:|---|---|
| PPT 与结构化汇报 | L3 | 腾讯训练营 25 页结课报告、索迪斯工作报告 | 需避免把目标值写成结果 |
| Excel 基础分析/数据透视 | L3 | 工单、查询单、月报和周报经历 | SQL、自动化分析未验证 |
| BRD/PRD | L3 | 自由笔记 BRD、打卡广场 PRD | 属个人方案，非上线产品 |
| 产品研究与方案设计 | L3 | AI 陪伴产品结课项目 | 用户研究原始 CSV 缺失 |
| AIGC 图片/视频生成 | L3 | 本人确认及作品集线索 | 需后续整理可公开作品链接 |
| 视频审美与多模态内容测评 | L3 | 音视频测评运营笔试、影视专业背景 | 尚无正式岗位结果数据 |
| 内容/活动运营拆解 | L3 | 原神 4.6 报告、美食品鉴及满意度活动 | 活动策略与商业结果归因需谨慎 |
| 活动现场执行与讲解 | L3 | 触达 800+、个人新增 65 人、80 份问卷 | 新增不等于长期留存或付费转化 |
| 客诉沟通与跨部门升级 | L3 | 残卫事件、客服借调 | 不可写成独立定责或独立闭环 |
| 工作报告、会议纪要、个人 SOP | L3 | 索迪斯月报/周报/工单流程 | 未证明组织级采纳 |
| 用户访谈/需求研究 | L2 | 客诉、现场交流、产品方案 | 正式访谈设计、招募、编码分析仍需训练 |
| 活动策划 | L2 | 参与线下活动、原神活动拆解 | 独立预算、排期、ROI 闭环证据不足 |
| 原型设计 | L2 | 本人确认需 AI/教程辅助 | 应补 Figma/Axure 可点击原型 |
| Agent/工作流搭建 | L1-L2 | 当前正在搭建求职 Agent | 尚未完成可运行 MVP 与评测集 |
| 终端操作 | L2 | 能在 AI/教程辅助下执行 CMD/PowerShell | 不主张独立编程或系统运维 |
| API、RAG、MCP、Tool Calling | L1 | 学习中 | 未有可运行证据前不写“熟练” |
| SQL/Python/统计建模 | L0-L1 | 暂无直接证据 | 不作为当前岗位卖点 |

## 3. 岗位适配结论

| 岗位方向 | 当前证据强度 | 结论 |
|---|---:|---|
| AIGC 内容运营 | 高 | 当前最容易用作品、影视背景和 AIGC 能力形成完整证据链 |
| AI 多模态内容测评/数据运营 | 高 | 音视频笔试与影视审美直接匹配；优先投非纯机械标注岗位 |
| AI 产品运营 | 中高 | 产品文档和 AI 研究较强；需要补“真实用户反馈—迭代—结果”项目 |
| AI 应用运营 | 中高 | 问题驱动学习与工具使用匹配；需要可运行 Agent/工作流案例 |
| 用户/活动运营 | 中 | 有活动、数据和客诉案例；缺成熟互联网指标与平台后台经验 |
| 普通内容运营 | 中 | 可胜任，但应优先选择能积累作品、数据或 AI 能力的岗位 |
| 客户成功 | 中低 | 服务与沟通有证据；若续费绩效、陌生开发或强销售则不匹配 |
| 纯产品经理 | 中低 | 有 PRD，但缺上线、研发协作和版本结果，优先产品助理/产品运营 |
| 纯技术/算法/开发 | 低 | 当前无代码与工程证据，直接淘汰有硬性编程要求的岗位 |
| 行政/纯剪辑 | 保底 | 能做但不作为主线；只有现金流紧张且主线漏斗持续无效时启用 |

## 4. 最短补证项目

1. 求职 Agent MVP：输入 JD，输出投递决策、简历动作、招呼语、面试准备，并保存决策日志。
2. 30 条 JD 评测集：人工金标与 Agent 决策对比，记录误杀、误投和理由一致率。
3. 多模态测评作品：选 10 条生成视频，建立审美、提示词一致性、时序和音画评分表。
4. 产品运营闭环：为自由笔记 PRD 做一个可点击原型和 5 名用户访谈，形成迭代记录。
5. 作品索引：把腾讯项目、自由笔记、原神报告、毕业设计和 AIGC 作品统一成可公开目录。
