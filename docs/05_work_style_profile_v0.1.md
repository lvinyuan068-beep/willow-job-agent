# 工作方式画像 v0.1

## 核心判断

Willow 并不排斥沟通或重复工作。真正影响可持续性的变量是：任务是否有明确目的、结果是否帮助到人、本人是否拥有合理角色与反馈、责任是否与权限匹配。

## 能量来源

- 一对一访谈，围绕具体观点或问题展开深入交流。
- 接收用户反馈，理解表面诉求背后的真实需要。
- 把用户问题转译成团队能够处理的信息。
- 与同事共同定位核心问题、提出方案并推动变化。
- 在跨部门合作中学习新的知识与工作方法。
- 产品／活动演示、清晰讲解服务和参与方式。
- 能看到任务对用户、团队或产品产生实际帮助。

## 可以接受的重复

- 可接受重复任务占比达到约 60% 或更高，前提是任务有目的、有结果、能观察变化。
- 对数据调研、工单检查、内容测评等重复流程保持开放。
- 倾向主动寻找更高效的方法，把重复流程优化为可复用方法。

## 主要消耗源

- 长时间没有明确职责、工作输入或结果反馈。
- 被视作临时劳动力，却承担不属于自身角色的最终责任。
- 责任大于权限，无法推动问题但需要承担后果。
- 团队没有入职介绍、角色说明、工作流程或带教反馈。
- 只做形式性服务、琐事或等待，无法理解任务价值。
- 沟通最后变成单向教育、打压或身份审视，问题没有推进。
- 依赖过度捧人、寒暄或权力距离的沟通环境。

## 沟通画像

- 不排斥陌生人，也不排斥客诉。
- 对“有问题—能沟通—能推进”的交流有动力。
- 对一对一用户访谈最感兴趣。
- 能接受产品演示、售前需求沟通。
- 能接受部分转化运营，例如群发活动、优惠券，但不接受直播卖课等模式。
- 与高职级人员沟通时容易预期被审视或忽略，需要通过结构化汇报模板和安全的团队环境降低负担。

## 对岗位筛选的影响

优先：用户研究、VOC、客户体验、AI 产品运营、AI 应用运营、模型测评与反馈闭环、客户成功中的需求分析板块。

谨慎：职责模糊的综合运营、纯执行外包、责任与权限不匹配的驻场岗位、强层级文化、没有带教的“培养岗”。

直接排除：陌生电话开发、直播卖课、以运营名义招聘销售、长期冷板凳式实习或无明确产出的岗位。

## 面试反向确认问题

1. 新人前三个月的具体职责、交付物和带教方式是什么？
2. 这个岗位对哪些结果负责，又拥有哪些推动权限？
3. 日常工作中分析、沟通、执行和重复审核大约各占多少？
4. 用户反馈如何进入产品／运营迭代流程？
5. 跨部门协作由谁发起，出现卡点时如何升级？
6. 团队怎样判断新人已经可以独立工作？
