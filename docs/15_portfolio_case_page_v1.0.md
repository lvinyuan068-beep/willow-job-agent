# Willow Job Agent｜从0到1搭建个人求职决策Agent

项目类型：AI产品设计 / Agent应用 / 个人项目  

项目状态：本地可运行，进入真实市场验证阶段  

我的角色：产品需求、Prompt架构、规则设计、证据审计、测试验收与迭代决策  

技术实现：AI辅助完成代码生成、调试与重构

## 一句话介绍

Willow Job Agent 是一套面向个人求职场景的本地AI决策工具：输入一份真实JD后，先判断是否值得投递，再生成证据安全的定向简历、HR话术、逐题面试答案和短期补齐方案。

## 01｜我发现了什么问题

求职时，真正消耗时间的并不只是“找岗位”，而是反复完成这些判断：

- 这个岗位是真运营还是销售转化？

- 我的学历、经验、城市和薪资是否符合？

- 应该使用哪版简历？

- 简历应该改什么，而不是只换几个关键词？

- HR主动联系时应该如何回复？

- 面试会问什么，怎样使用真实经历回答？

- 哪些能力可以短期补齐，哪些属于硬门槛？

- AI生成的简历数据是否真实可靠？

普通大模型能够给建议，但不同对话的判断口径容易变化，也可能把未经核验的数据写成正式成果。

因此，我把目标定义为：建立一个有固定规则、有个人证据库、可以测试和持续迭代的求职Agent。

## 02｜产品方案

系统输入：

- 岗位名称与公司；

- 城市、薪资和工作地点；

- 完整JD；

- HR沟通内容；

- 公司或岗位补充信息。

系统优先输出：

1. 投不投；

2. 是否修改简历；

3. 可直接复制的HR沟通话术；

4. 岗位定向面试问题与示例答案；

5. 短期可补齐技能和执行步骤。

同时输出岗位评分、真实职责分类、风险项、缺失信息、证据来源和下一步行动。

## 03｜核心功能

### 岗位筛选

系统将岗位分为：

- 可以直接投递；

- 修改简历后投递；

- 补充信息后决定；

- 直接淘汰。

能够识别销售、陌生开发、无薪试岗、收费培训、明显经验门槛和低于生活底线的薪资。

### 证据安全

我将个人经历拆成16个可复用证据块，并设置10项禁止表述。

例如：

- 可以使用“审核2000+条工单”；

- 可以使用“活动触达800+”；

- 可以使用“个人新增65名活动用户微信”；

- 不使用缺少底表的65%到32%精确KPI；

- 不把个人流程写成公司正式落地SOP；

- 不把团队事件写成个人独立定责和闭环。

### 定向简历

系统不只告诉用户“简历需要修改”，还生成完整的岗位定向正文。

当前支持CRM、AIGC内容、AI产品、AI数据或多模态测评、AI应用及普通运营方向。

### 面试执行包

每道问题包含：

- 示例答案；

- 使用证据；

- 可能追问；

- 注意事项；

- 答案质量状态。

当前已针对CRM和AIGC内容岗位建立专项题库，并加入语义去重和答非所问检查。

### 联网研究与模型主备

- Kimi作为主模型；

- DeepSeek作为备用模型；

- Tavily检索公司公开网页资料；

- 本地规则引擎提供可配置的降级模式。

Tavily只能提供公开网页线索，不替代官方工商核验。

## 04｜一次关键迭代

在真实AIGC内容生产岗位测试中，初始版本出现两个问题：

- 面试答案过于通用；

- 部分答案残留CRM岗位表达。

当时8道答案中只有3道通过质量检查，通过率为37.5%。

我随后进行了以下调整：

1. 单独建立AIGC问题意图；

2. 增加作品介绍、分镜拆解、质量迭代、版本记录和内容数据复盘答案；

3. 增加岗位类型隔离，防止CRM答案进入AIGC岗位；

4. 增加证据、追问和能力边界；

5. 重新运行真实岗位评测。

修改后，同一岗位8道答案全部通过质量检查，真实岗位端到端评测从1/2提升为2/2。

这里的37.5%和100%是项目内部答案质量评测结果，不代表真实面试通过率。

## 05｜当前验证结果

截至2026年7月17日：

- 本地规则评测：8/8通过；

- 模型故障切换：3/3通过；

- 网页端回归测试：7/7通过；

- 初始真实岗位评测：2/2通过；

- CRM岗位面试答案：8/8通过；

- AIGC岗位面试答案：8/8通过。

真实岗位样本目前只有两个，尚不足以代表整体市场准确率。

## 06｜我的具体贡献

我负责：

- 通过多轮访谈梳理真实求职问题；

- 确定用户画像、岗位优先级和求职底线；

- 设计Agent工作流和五项核心输出；

- 整理个人经历和证据边界；

- 定义风险岗位淘汰规则；

- 设计简历、面试和补齐任务的验收标准；

- 使用真实岗位进行测试；

- 根据错误结果确定下一轮迭代优先级；

- 逐项运行类型检查和回归测试。

AI协助完成TypeScript代码生成、错误修复和模块重构。

因此，我会将该项目描述为“AI辅助搭建”，而不是“独立编程开发”。

## 09｜下一步

后续每获得一份完整真实JD，就记录：

- Agent判断；

- 实际投递动作；

- HR是否查看；

- 是否获得有效回复；

- 是否获得面试；

- 最终结果；

- 判断错误及改进方案。

项目第一阶段的最终成功指标不是测试全部通过，而是获得多个符合目标方向的面试机会，或成功取得一个符合底线要求的Offer。

## 项目结论

Willow Job Agent 已从一段求职Prompt发展为一个具备岗位筛选、双模型主备、联网研究、证据保护、定向简历、面试答案、网页交互和回归评测的本地AI产品原型。

它当前已经能够辅助真实求职，但仍需要通过更多真实岗位和招聘结果持续验证。
