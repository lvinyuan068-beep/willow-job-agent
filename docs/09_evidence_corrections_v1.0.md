# 09｜求职证据纠偏台账 v1.0

更新日期：2026-07-15  
用途：约束简历、招呼语、面试回答和 Agent 判断，禁止把推测、目标或 AI 概括写成已验证成果。

## 1. 证据等级

| 等级 | 定义 | 使用规则 |
|---|---|---|
| A | 本地原始文件可直接验证，且内容与本人表述一致 | 可直接用于简历；仍需准确说明个人角色 |
| B | 本人确认，且有本地材料或上下文间接佐证 | 可使用，但采用保守表述，不扩大因果 |
| C | 本人口述，暂无可核验材料 | 可用于面试案例；简历使用前应补证据 |
| D | AI 概括、目标值、公开资料推导或存在冲突 | 不得作为已实现成果；只能标为方案、观察或待核验 |

## 3. 残卫事件的面试边界

推荐结构：

1. 情境：受伤员工因残卫扶手松动再次摔倒，投诉涉及安全、设施、HR 与员工部门。
2. 任务：客服需在一个工作日内响应，但技术原因和责任方无法立即确认。
3. 行动：线下了解诉求、安抚情绪、同步调查进度、陪同就医流程、拉通设施及相关负责人、持续跟进约两周。
4. 结果：设施完成排查与加固，员工进入工伤流程；但设施对外邮件未按原协商发送，客服后续停止代替责任部门推进。
5. 复盘：客服能负责响应、沟通和升级，不能替责任部门定责；跨部门任务必须明确负责人、截止时间和对外口径。

## 4. 使用规则

- 所有数字先查本台账；找不到证据时降级为范围、观察或定性描述。
- “参与、负责、独立完成、主导、推动落地”必须严格区分。
- 公开资料分析只能证明研究与判断能力，不能证明业务结果。
- Agent 生成简历时必须同时返回：证据等级、来源文件、是否需补证。
